##### Docker

> "你，是不是也遇到过’在我电脑上能跑’的问题？"  
> ——AptS:1547（手里拿着 Dockerfile 准备糊你脸）

天才的发明：
- **隔离各个环境**：以前在自己电脑上都很随意，软件装就装了，然后 C 盘就逐渐爆满，你还不知道是谁的锅。在服务器上就不一样了，强迫症这一块，每个服务必须要一个容器，一切井然有序，井水不犯河水。这块是你的，那块是你的，配置好一次 `compose.yaml` 然后 up 就开 down 就删，超级省事啊。迁移也随便挪，备份只需要压缩那一个目录就好了，太有生活了。
- **把应用和环境打包在一起，保证 "开箱即用"**：自己的程序写好 Dockerfile 就一定能在其他地方跑，别人的程序只要把他们给的镜像拉下来就一定能跑，只需要思考个人配置就好了，太有生活了。

非常感谢 47 的安利，也很感谢它当时帮我那个粗糙的网页增加 Docker 支持，要不然估计我回头把它塞上服务器真的会头疼到死。

> 突然莫名觉得有点像 iOS 系统的 App 逻辑怎么回事，干净整洁这一块。

### GitHub

还记得以前初中的时候私有仓库还收费，大学之后一看原来已经完全免费了，物是人非（bushi）。  
今年也是开始高强度用 GitHub 了，初中那会还在苦恼 GitHub 有什么用来着（因为当时脸皮薄不敢把自己代码开源，现在想想实际上也没啥）。

- **Star！：** 发现好玩的项目必点 Star，收藏夹里躺满了“以后一定看”的各种小玩意。
- **Actions：** 慢慢开始用 Action 构建/发布自己的项目，DRY(Don't Repeat Yourself)，能自动化的肯定不会自己再动手，~~虽然写 Action 可能反而花更多时间~~。
- **Fork：** 发现一个好玩的东西，但是它少了一个我想要的功能怎么办？ Fork 别人仓库，自己写😋开源魅力时刻。
- **PR：** 偶尔也会找机会给别人贡献代码，虽然不多啦，虽然自己水平也不算好，但是还是会继续尝试的，这才是开源社区的意义嘛。
- **Force Push：** 偶尔发现自己写错一两个东西，悄悄强制推送，反正我的仓库没人和我协作（

> [YewFence On GitHub](https://github.com/YewFence/) 欢迎来看我最近又新写了什么东西😁

### 自托管服务：Welcome to YewYard

好玩的东西出现了，我的数据我做主，开源社区有超级多好东西，以前都没发现。

- **VaultWarden：** 密码管理器要交钱，不想交怎么办？自己部署一个在服务器上呗。
- **Cloudreve：** 网盘个个都要登录/充会员才能下载怎么办？搭一个自己的网盘，我爱干嘛干嘛。
- **Matrix：** 不想让自己的聊天记录被厂商管控，于是自己搭了一个 Matrix 服务器，~~虽然没人用~~。
- **Gitea：** 不想在 GitHub 上给自己的历史记录织毛衣，但是又想备份自己的完整 Git 历史，那就自己托管一个 Git 云平台吧。

然后就是给我的各个服务起名字，给我的词汇量提出了巨大的挑战🥲Gemini 救我。最后定下来一个习惯：我的服务/项目都是 Yew 开头，取各个相关的意象，清晰而有个人风格，像是 YewTrack，YewResin，YewGrove 之类的。
